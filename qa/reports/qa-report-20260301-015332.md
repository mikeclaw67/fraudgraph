## Top Issues vs Palantir Reference

### 🔴 CRITICAL
1. **No row interaction affordances** — rows show no hover state, no click target, no expand icon, no primary action button. Queue is read-only, not a triage workstation.
2. **No bulk selection** — no checkboxes, no multi-select. Can't triage multiple rings at once.

### 🟠 HIGH
3. **KPI cards lack containers/context** — bare numbers floating with no card borders, no trend deltas, no sparklines. Unreviewed count (9) is orange but $66.9M exposure has zero color treatment.
4. **Row height is excessive** — TYPE badge pills are two-line, inflating row height to ~50px. Only 8-9 rows visible above fold instead of 15-20. Critically low data density for a triage queue.
5. **No search/filter beyond status tabs** — can't filter by investigator, type, date range, exposure range. Status tabs have no counts ("NEW" should show "NEW (9)").

### 🟡 MEDIUM
6. **Inconsistent badge systems** — TYPE badges are large multi-line pills with icons; STATUS badges are tiny flat rectangles. Two completely different visual languages in the same table.
7. **"SMOKING GUN" truncates** — column too narrow, key identifiers cut off with ellipsis. "DETECTED" dates wrap to two lines (too narrow or format too long).
8. **No sort affordances** — only EXPOSURE column shows sort indicator. RISK, MEMBERS, DETECTED are non-interactive (or appear so).

### 🟢 LOW
9. **"5 Issues" floating badge** (bottom-left) — looks like a debug artifact. Unexplained, visually disconnected from queue content.
