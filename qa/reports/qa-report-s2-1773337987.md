## ❌ BUG-001 [CRITICAL] — Open Case crashes Ring Detail page

**Component:** `ring-detail.tsx` line 1006  
**Trigger:** Click "Open Case" on any Ring Detail  
**Error:** `Runtime TypeError: Cannot read properties of undefined (reading 'length')`

**Root cause:**
```tsx
// Line 1006 — no null guards
{caseData.audit_trail.length + caseData.notes.length} events
```
When `openCase()` creates a new case, `audit_trail` and/or `notes` may be `null` or `undefined`. The component crashes before EvidenceChecklistPanel or ReviewerPanel ever render.

**Reproduction steps:**
1. Navigate to `/rings/ring_001`
2. Click "Open Case" button
3. → Next.js error overlay: `Cannot read properties of undefined (reading 'length')`

**Fix (1 line):**
```tsx
{(caseData.audit_trail?.length ?? 0) + (caseData.notes?.length ?? 0)} events
```

**Impact:** The entire S2 feature set (EvidenceChecklistPanel, ReviewerPanel) is gated behind "Open Case" — this crash means neither panel can be tested in production. Core S2 deliverable is unreachable.
